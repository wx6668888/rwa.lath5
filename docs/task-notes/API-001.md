# API-001 后端骨架验收备注

**状态：** 待审核  
**完成日期：** 2026-07-11  
**设计前置：** 已在开工前完整阅读 `DESIGN.md`。本任务是基础服务，不新增用户界面或概念图；它为后续完整页面的数据、状态与异常处理提供 API 承载层。

## 本次完成内容

- 新建独立 NestJS 服务：`apps/api/`。
- 提供版本化 API 前缀：`/v1`。
- 提供 `GET /v1/health` 与 `GET /v1/health/ready`。
- 配置 OpenAPI：`GET /v1/docs` 与 `GET /v1/docs-json`。
- 每个响应生成或透传 `x-request-id`，并在健康响应体返回同一 `requestId`。
- 统一异常格式：`error.code`、`error.message`、`requestId`、`path`、`timestamp`。
- 启用 CORS 白名单、安全响应头、全局输入验证和非白名单字段拒绝策略。
- 增加 `.env.example` 与本地运行说明。

## 验证

- 已执行 `npm run lint`（TypeScript 无错误），通过。
- 已执行 `npm run build`（Nest build），通过。
- 本地 API 运行于 `http://localhost:4000`。
- `GET /v1/health` 与 `GET /v1/health/ready` 均返回 200 和一致的请求 ID 头/体。
- `GET /v1/docs-json` 返回 200。
- 不存在的 `/v1/missing` 返回 404，且符合统一错误格式。

## 后续边界

- 当前没有数据库、身份、KYC、钱包、订单或 Polymarket 调用；这些能力分别由 `DB-001`、`API-002` 至 `API-010` 实现。
- Swagger 当前只展示系统健康接口；领域模块加入后必须同步补充 DTO、鉴权和 OpenAPI 描述。
