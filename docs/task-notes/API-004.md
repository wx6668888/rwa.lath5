## 实现内容

新增 `apps/api/src/compliance/` 模块，注册进 `AppModule`：

1. **实体（映射 DB-002 四表）**：`kyc-case.entity.ts`、`eligibility-profile.entity.ts`、`risk-flag.entity.ts`、`screening-case.entity.ts`，schema 为 `app`。
2. **PARTNER-001 类型化桩适配器**：
   - `kyc-provider.interface.ts`：`KycProvider` 接口（submitCase / getCase）。
   - `sanctions-provider.interface.ts`：`SanctionsProvider` 接口（screen，覆盖 sanctions/pep/adverse_media/wallet_risk）。
   - `stub-kyc.provider.ts`：`StubKycProvider` 原型桩，返回 `stub:<userId>:<uuid>` 引用，不发起真实外部调用。
   - `stub-sanctions.provider.ts`：`StubSanctionsProvider` 原型桩，默认 `clear`，命中 `SANCTIONS_BLOCKLIST` 环境变量则返回 `confirmed_match`。
   - `region-policy.ts`：`RegionPolicy` 从 `ALLOWED_REGIONS` 读取白名单（默认 `ALL` 允许全部），地域准入判定集中于此。
3. **ComplianceService（核心）**：
   - KYC 生命周期：`startKyc`（not_started→in_progress）、`submitKyc`（调用 KycProvider，加密存储供应商引用，submitted）、`decideKyc`（approved/rejected，自动触发资格重算）。
   - 制裁/筛查：`screen(userId, kind, identifiers)` 调用 SanctionsProvider，落 `screening_cases`；confirmed_match 自动开高风险标记。
   - **统一资格评估 `evaluateEligibility(userId, productScope, region?)`**：聚合 KYC 状态、制裁筛查结果、地域准入与开放风险标记，产出决策 `browse_only | eligible | ineligible | manual_review`，幂等（同 user+policy_version+product_scope 更新 `eligibility_profiles`）。这是 API-008 订单侧将复用的同一入口。
   - 风险标记：`openRiskFlag` / `resolveRiskFlag`（open/under_review/resolved/dismissed）。
   - 查询：`getKycStatus` / `getEligibility` / `listScreening`。
4. **ComplianceController**：`/v1/compliance/*` 路由（KYC start/submit/decision、screening、eligibility evaluate/get、kyc status、risk-flags 开/解）。
5. **契约扩展**：`packages/contracts` 新增 `COMPLIANCE_ERROR_CODES`（KYC_CASE_NOT_FOUND / KYC_ALREADY_SUBMITTED / KYC_NOT_SUBMITTED / ELIGIBILITY_NOT_FOUND / SCREENING_NOT_FOUND / REGION_BLOCKED / RISK_FLAG_NOT_FOUND / PROVIDER_ERROR）。
