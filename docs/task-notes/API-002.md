# API-002 · 身份与账户（Identity & Account）

**状态：** 完成
**负责人：** Hermes（接手原计划 Claude Code 的里程碑）
**完成日期：** 2026-07-11
**依赖：** API-001（骨架）、DB-001（连接）、DB-002（身份合规表）

## 范围
实现身份与账户闭环：用户、邮箱注册/验证、OAuth（google/x）、钱包签名挑战（EIP-191）、会话签发、账户恢复、会话撤销。

## 交付物
- `apps/api/src/identity/identity.module.ts` — Nest 模块（注册于 `app.module.ts`）
- `identity-crypto.service.ts` — HMAC 标识符哈希、AES-256-GCM 加密、EIP-191 钱包验签、nonce/会话 token 生成、无状态签名 token
- `user.entity.ts` / `login-identity.entity.ts` / `device.entity.ts` / `session.entity.ts` — 映射 DB-002 的 `app.users/app.login_identities/app.devices/app.sessions`
- `dto/*` — class-validator DTO（注册、验证、钱包挑战/验签、OAuth、恢复）
- `identity.service.ts` — 业务逻辑：注册冲突（409）、邮箱验证、钱包挑战+验签开会话、OAuth 幂等登录、恢复（防枚举）、会话撤销
- `identity.controller.ts` — `POST /v1/auth/*`（register/verify-email/wallet/challenge/verify/oauth/:provider/recover/logout）

## 验收（对照 Task-Board 四项）
- [x] **nonce** — `createWalletChallenge` 生成一次性 nonce（内存存储 + TTL，原型级；生产需 Redis，见 INFRA-002）
- [x] **会话** — 钱包/OAuth 成功后签发 `sessions` 记录 + 设备指纹，返回 Bearer token（HMAC token_hash，明文不下库）
- [x] **绑定冲突** — 邮箱已验证重复注册返回 `IDENTITY_EMAIL_TAKEN`；错误 nonce/签名返回 401；OAuth 已绑定则幂等登录
- [x] **恢复** — `recover` 对已验证邮箱返回签名恢复 token，未知邮箱返回通用响应（不泄露枚举）

## 安全要点
- 标识符与 PII 均经 HMAC/AES 处理，明文不落库（密钥来自 `IDENTITY_HMAC_KEY`/`IDENTITY_ENC_KEY`，见 `.env.example`）
- 钱包验签用 @noble/curves 的 secp256k1 + keccak，纯 TS 无原生依赖
