## 实库验收记录

本机 Ubuntu 24.04 WSL 安装 PostgreSQL 16.14，并以与 `.env.example` 一致的本地账号建立 `rwa_lat_dev` 与 `rwa_lat_test`：

1. 开发库首次 `npm run db:migration:run` 创建 `schema_migrations`、`app` schema 与 1 条迁移记录。
2. `npm run db:migration:revert` 删除 `app` schema 和迁移记录，事务提交成功。
3. 再次 `npm run db:migration:run` 恢复 schema 和迁移记录；第三次执行显示无待处理迁移。
4. 使用 `NODE_ENV=test` 与 `TEST_DATABASE_URL` 对测试库独立执行迁移成功。
5. PostgreSQL 查询结果为 `rwa_lat_dev|1` 与 `rwa_lat_test|1`，确认迁移历史物理隔离。

容器实际挂载的 `001-create-test-database.sql` 已对全新测试库执行，并连续运行两次验证幂等。Docker Compose 运行时本身尚未在本机执行，因为本机没有 Docker；该运行时兼容性继续作为基础设施环境检查，不阻止数据库迁移框架进入审核。
